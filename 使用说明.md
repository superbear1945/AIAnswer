## 0. 必填项一览（先看这里）

打开软件后点击「**设置**」，按你选择的语音识别方式，在对应位置填入以下内容即可使用。

> 你也可以直接编辑 `config/user.json` 文件填入，效果相同。

### 如果选「讯飞语音听写 (xunfei)」

| 填在哪里 | 填什么 | 说明 |
|----------|--------|------|
| 设置 → 讯飞 → APPID | 你的讯飞 APPID | 一串数字 |
| 设置 → 讯飞 → APIKey | 你的讯飞 APIKey | 一串字母数字 |
| 设置 → 讯飞 → APISecret | 你的讯飞 APISecret | 一串字母数字 |

### 如果选「兼容音频转写 API (http_asr)」

| 填在哪里 | 填什么 | 说明 |
|----------|--------|------|
| 设置 → HTTP ASR → API Base | 语音转写 API 地址 | 例如 `https://api.openai.com/v1` 或兼容地址 |
| 设置 → HTTP ASR → API Key | 访问密钥 | 服务商提供的 Key；若未鉴权可留空 |
| 设置 → HTTP ASR → Model | 模型名 | 例如 `whisper-1`，一般默认即可 |

### LLM（大语言模型，二选一必填）

| 填在哪里 | 填什么 | 说明 |
|----------|--------|------|
| 设置 → LLM → API Base | 大模型服务地址 | 例如 `https://api.deepseek.com/v1` |
| 设置 → LLM → API Key | 模型访问密钥 | 从对应平台获取 |
| 设置 → LLM → Model | 模型名称 | 例如 `deepseek-chat` |

## 1. 快速开始

### 前置条件

- 电脑具备图形界面（Windows / macOS / Linux 桌面环境）
- Python 3.10 或更高版本
- 麦克风（笔记本自带或外接）
- 能访问互联网的上课环境

**建议**：
- Windows 用户优先准备 **Python 3.11 或 3.12**；本项目当前在这两个版本上更稳。
- 如果你的系统默认 Python 是 **3.14**，建议优先使用下方 **方式 B：uv**，或在方式 A 中显式指定 3.11/3.12。

**如何自查系统是否满足基本条件**：

```bash
# Windows PowerShell
python --version
py -0p
where python

# macOS / Linux
python3 --version
which python3
```

判断标准：
- 能看到 Python 版本号，说明系统已装 Python。
- 若是 Windows，`py -0p` 能列出 `3.11` 或 `3.12`，说明你可以按推荐版本创建环境。
- 若命令提示不存在，请先安装 Python，再继续下面步骤。

> **Linux 用户注意**：若后续安装 `pyaudio` 报错，请先安装系统依赖：
> ```bash
> # Ubuntu/Debian
> sudo apt-get install python3-tk portaudio19-dev
> # Fedora
> sudo dnf install python3-tkinter portaudio-devel
> ```

---

### 方式 A：标准 venv（推荐大多数用户）

**方式 A 前置条件**：
- 系统里已经安装了可用的 Python。
- Windows 用户建议准备 **Python 3.11 / 3.12**；如果只有 3.14，建议改走方式 B。
- Python 自带 `venv` 模块可用。

**如何检查是否满足方式 A 条件**：

```bash
# Windows PowerShell
python --version
py -0p
python -m venv --help

# macOS / Linux
python3 --version
python3 -m venv --help
```

判断标准：
- `python -m venv --help` 或 `python3 -m venv --help` 能正常输出帮助，说明可以创建虚拟环境。
- Windows 下如果 `py -0p` 里有 `3.11` 或 `3.12`，优先使用该版本建环境。

```bash
# 1. 进入项目目录
cd AIAnswer

# 2. 创建并激活虚拟环境
# Linux / macOS
python3 -m venv .venv
source .venv/bin/activate

# Windows CMD（若默认 python 已是 3.11/3.12）
python -m venv .venv
.venv\Scripts\activate.bat

# Windows PowerShell（若默认 python 已是 3.11/3.12）
python -m venv .venv
.venv\Scripts\Activate.ps1

# Windows PowerShell（若装了多个 Python，推荐显式指定 3.11）
py -3.11 -m venv .venv
.venv\Scripts\Activate.ps1

# 3. 安装依赖（仅需一次）
pip install -r requirements.txt

# 4. 运行程序
python main.py
```

**以后每次上课**：
```bash
cd AIAnswer

# Linux / macOS
source .venv/bin/activate

# Windows PowerShell
.venv\Scripts\Activate.ps1

python main.py
```

---

### 方式 B：uv（推荐追求速度的用户）

[uv](https://github.com/astral-sh/uv) 是新一代高性能 Python 包管理器，比 pip 快 10~100 倍，内置虚拟环境管理。

**方式 B 前置条件**：
- 系统里已安装 `uv`，或者可以联网安装 `uv`。
- 首次创建环境和安装依赖时需要联网。
- 若本机没有合适的 Python 版本，`uv` 也可以帮你下载并创建指定版本的环境。

**如何检查是否满足方式 B 条件**：

```bash
# Windows / macOS / Linux 通用
uv --version
uv python list
```

判断标准：
- `uv --version` 能输出版本号，说明 `uv` 已可用。
- `uv python list` 能列出可用或可下载的 Python 版本；建议优先使用 `3.11` 或 `3.12`。

**Step 1 — 安装 uv**：
```bash
# 通过 pip（最简单）
pip install uv

# 或独立安装（不依赖系统 Python，推荐长期用）
# Linux/macOS:
curl -LsSf https://astral.sh/uv/install.sh | sh
# Windows:
# powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**Step 2 — 创建环境并安装依赖**：
```bash
cd AIAnswer

# 推荐显式指定 3.11，避免不同机器默认选到不兼容版本
uv venv .venv --python 3.11
uv pip install -r requirements.txt
```

**Step 3 — 运行程序**：
```bash
# uv 会自动识别 .venv，无需手动 source activate
uv run python main.py
```

> 💡 **uv 的优势**：
> - 安装依赖速度极快（编译好的 wheel 直接缓存）
> - `uv run` 自动使用 `.venv`，不用记 activate 命令
> - 与标准 venv 完全兼容，不会锁定你进 uv 生态

## 3. 配置详解

点击「设置」后，弹出配置对话框，共包含 4 个板块。

### 3.1 ASR 提供商（语音识别）

下拉框可选择：

- **`xunfei`** — 讯飞语音听写（WebSocket 流式）
- **`http_asr`** — 项目内置的通用音频转写客户端（兼容 OpenAI Whisper API 格式）

**选择后，下方只会显示对应提供商的配置项。**

---

#### 3.1.1 讯飞语音听写 (xunfei)

| 配置项 | 说明 | 获取方式 |
|--------|------|----------|
| **APPID** | 讯飞应用标识（8位十六进制） | 控制台 → 语音听写服务页 → Websocket服务接口认证信息 |
| **APIKey** | 接口密钥（32位） | 同上 |
| **APISecret** | 接口密钥（32位） | 同上 |

**特点**：
- 真·流式识别，延迟最低（200ms ~ 1s）
- 中文课堂场景识别率极高
- 新用户有**免费额度**，一般上课使用几乎不花钱

**适用场景**：追求低延迟、上课环境网络稳定。

---

#### 3.1.2 兼容音频转写 API (http_asr)

这是项目内置的语音转写客户端，兼容任何提供 `POST /v1/audio/transcriptions` 接口的服务；你**不需要额外自建一个 HTTP ASR 中转服务**。

| 配置项 | 说明 | 示例 |
|--------|------|------|
| **API Base** | 服务根地址 | `https://api.openai.com/v1` |
| **API Key** | 访问密钥 | `sk-xxxxxxxx` |
| **Model** | 模型名称 | `whisper-1` |
| **Language** | 语言代码 | `zh`（中文） |
| **分段时长(ms)** | 每段音频长度 | `1500`（1.5秒） |

**特点**：
- 兼容面广，支持**自建 Whisper / FunASR** 等开源服务
- 支持 OpenAI 官方 Whisper API
- 延迟略高于讯飞（约 1~3 秒额外延迟，因为采用分段上传）
- 适合直接接云端转写 API、已有自建服务、或不便申请讯飞账号的用户

**适用场景**：希望直接使用 OpenAI 兼容语音转写 API，或需要将语音数据留在自己服务器上。

**重要说明**：
- 只要目标平台支持 `/audio/transcriptions`，你在软件里填 API Base、API Key、Model 就能直接用。
- 如果某个平台只有聊天模型接口（如 `/chat/completions`），没有音频转写接口，那它不能直接承担 ASR。

**自建服务示例**：
若你使用 [openai-whisper-asr-webservice](https://github.com/ahmetoner/openai-whisper-asr-webservice) 或类似项目部署了本地 Whisper 服务，假设地址为 `http://localhost:9000/v1`，则：
- API Base 填 `http://localhost:9000/v1`
- API Key 留空或填任意占位符（视你的服务是否鉴权）
- Model 填 `whisper-1`（兼容占位）

---

### 3.2 大语言模型 (LLM)

用于生成提问的回答。支持任何 **OpenAI 兼容格式** 的 API。

| 配置项 | 说明 | 推荐值 |
|--------|------|--------|
| **API Base** | 模型服务端点 | 见下表 |
| **API Key** | 模型访问密钥 | 从对应平台获取 |
| **Model** | 模型名称 | 见下表 |

**国内常用方案**：

| 服务商 | API Base | 推荐模型 |
|--------|----------|----------|
| **DeepSeek** | `https://api.deepseek.com/v1` | `deepseek-chat` |
| **阿里通义** | `https://dashscope.aliyuncs.com/compatible-mode/v1` | `qwen-turbo` / `qwen-max` |
| **OpenAI** | `https://api.openai.com/v1` | `gpt-4o-mini` |
| **月之暗面** | `https://api.moonshot.cn/v1` | `moonshot-v1-8k` |

> 💡 **省钱建议**：课堂回答不需要最强模型，`deepseek-chat` 或 `gpt-4o-mini` 级别完全够用，费用极低。

**系统提示词（不可改）**：
```
回答该提问，口语化并简短精炼
```

---

### 3.3 提问触发词

每行填写一个触发词。当语音识别结果中包含这些词时，即视为老师提出了问题，自动触发 AI 回答。

**默认触发词**：
```
问个问题
请一位同学
谁来回答
怎么看
有什么想法
思考一下
对吗
是不是
请回答
哪位同学
你的观点
如何理解
怎么看待
有什么见解
举例说明
举例来讲
```

**自定义建议**：
- 如果你的老师有口头禅（如"考考你们""谁来试试"），可以直接添加。
- 不需要加标点，纯文本匹配即可。
- 注意：如果触发词太常见（如"的"），会导致频繁误触发，请避免。

## 4. ASR 提供商选择与对比

| 对比项 | 讯飞 (xunfei) | 通用 HTTP (http_asr) |
|--------|---------------|----------------------|
| **协议** | WebSocket 流式 | HTTP 分段上传 |
| **延迟** | 极低 (~0.5s) | 中等 (~2~4s) |
| **中文准确率** | 极高（课堂优化） | 取决于后端模型 |
| **是否需要自建** | 否 | 可选（也可用 OpenAI 官方） |
| **数据隐私** | 发送至讯飞云端 | 可发送至自建服务器 |
| **费用** | 新用户免费额度充足 | OpenAI 按量计费 / 自建免费 |
| **网络要求** | 需访问 `iat-api.xfyun.cn` | 需访问你填写的 API Base |

**如何选择？**
- 上课追求低延迟、不想折腾 → **选讯飞**
- 想直接接兼容音频转写 API、或有自建 Whisper 服务 → **选 http_asr**
- 不确定 → **先选讯飞**，不好用再切换

## 5. 首次配置向导

### 步骤 A：申请讯飞 ASR（如选 xunfei）

#### A-1 注册账号

1. 访问 [讯飞开放平台](https://www.xfyun.cn/)
2. 点击右上角「登录/注册」，完成账号注册
3. 首次使用需完成**实名认证**（个人认证即可）

#### A-2 创建应用

1. 登录后，进入 [讯飞控制台](https://console.xfyun.cn/app/myapp)
2. 点击「**创建新应用**」按钮
3. 填写：
   - **应用名称**：随意填写，例如 `课堂助手`
   - **应用分类**：选择「应用 → 教育学习 → 学习」（或任选）
   - **应用功能描述**：简要描述即可
4. 点击「提交」，创建成功后会获得一个 **APPID**（8位，如 `5465e2d2`）

#### A-3 开通语音听写服务

1. 在控制台顶部菜单栏，展开「**语音识别**」→ 点击「**语音听写**」
   - 也可直接访问 [语音听写服务页](https://console.xfyun.cn/services/iat)
2. 页面会显示当前应用（即你刚创建的应用）的服务信息
3. 确认「**普通话**」行的状态为「**已开通**」（新用户有免费额度，开通即用）
4. 如果显示「未开通」，点击该行的「试用」或「购买」按钮开通

#### A-4 获取 APPID / APIKey / APISecret

1. 在语音听写服务页（`https://console.xfyun.cn/services/iat`）向下滚动
2. 找到 **「Websocket服务接口认证信息」** 区块，页面中会显示三行：
   - **APPID**：8位十六进制字符串（如 `5465e2d2`）
   - **APISecret**：32位字符串（如 `MWRhNTM2M2RjOTY5OThlZGM3YmFiZTM2`）
   - **APIKey**：32位字符串（如 `c1164176527a32750423023bbf27e6ba`）
3. 将这三项信息复制下来，填入软件设置

> ⚠️ **注意**：APISecret 和 APIKey 仅在服务详情页可见，不在「我的应用」编辑页中。如果页面未显示认证信息，请确认你已从「语音识别 → 语音听写」进入，而不是从「我的应用 → 编辑」进入。

#### A-5 在软件中填入凭证

1. 打软件 → 点击「**设置**」
2. ASR 提供商下拉框选择 **`xunfei`**
3. 在「讯飞语音听写」区块填入：
   - **APPID**：粘贴 A-4 步骤获取的 APPID
   - **APIKey**：粘贴 A-4 步骤获取的 APIKey
   - **APISecret**：粘贴 A-4 步骤获取的 APISecret
4. 点击「**保存**」

### 步骤 B：申请 LLM API（任选一家）

**以 DeepSeek 为例**：
1. 访问 [DeepSeek 开放平台](https://platform.deepseek.com/)
2. 注册账号，创建 API Key
3. 在软件「设置」→ LLM 板块：
   - API Base 填 `https://api.deepseek.com/v1`
   - API Key 填入刚创建的 Key
   - Model 填 `deepseek-chat`

**以阿里通义为例**：
1. 访问 [阿里云百炼](https://bailian.console.aliyun.com/)
2. 创建 API Key
3. 在软件「设置」→ LLM 板块：
   - API Base 填 `https://dashscope.aliyuncs.com/compatible-mode/v1`
   - API Key 填入刚创建的 Key
   - Model 填 `qwen-turbo`

### 步骤 C：保存并测试

1. 点击「保存」
2. 回到主界面，点击「开始听课」
3. 对着麦克风说话，观察左侧字幕区是否出现文字
4. 说一句包含"问个问题"的话，观察右侧是否出现 AI 回答

## 7. 常见问题（FAQ）

### Q1：点击「开始听课」后提示"无法连接讯飞 ASR"

- 检查 APPID / APIKey / APISecret 是否填写正确（不要有多余空格）
- 确认三项凭证是从「语音听写服务页」的 **Websocket服务接口认证信息** 区域获取的，而非应用编辑页
- 确认网络能访问 `wss://iat-api.xfyun.cn`
- 确认讯飞控制台中「语音听写（流式版）」服务已开通（普通话行状态为「已开通」）
- 若在校内网，可能需要关闭代理或开启特定端口

### Q2：字幕区有文字，但右侧不触发 AI 回答

- 确认你说的话包含了触发词（如"问个问题"）
- 检查 LLM 的 API Key 和 API Base 是否填写正确
- 查看底部状态栏是否有 LLM 错误提示（401 = Key 无效，429 = 请求过快）
- 尝试在「设置」→ 触发词中添加更多匹配词

### Q3：AI 回答很慢，甚至超时

- 国内用户优先使用 DeepSeek / 阿里通义，避免跨国网络延迟
- 检查教室 WiFi 是否稳定
- 若使用 http_asr，可将「分段时长」调小至 1000ms，但会增加 API 调用次数

### Q4：字幕重复、跳动、或整行删除重写

- **这是流式 ASR 的正常现象**。讯飞会实时修正前面识别错的字，软件已做优化处理。
- 若跳动严重，检查麦克风是否有杂音，或尝试外接 USB 麦克风。

### Q5：可以保存课堂记录吗？

- 当前版本暂未内置导出功能。
- 可手动复制左侧「实时字幕」区域内容，粘贴到笔记软件中保存。
- 后续版本计划支持一键导出 Markdown。

### Q6：如何切换 ASR 提供商？

- 点击「设置」→ 顶部「ASR 提供商」下拉框 → 选择 `xunfei` 或 `http_asr`
- 下方配置表单会自动切换为对应参数
- 保存后重新点击「开始听课」即可生效

### Q7：http_asr 可以用哪些服务？

- **OpenAI 官方 Whisper API**：`https://api.openai.com/v1`
- **自建 Whisper 服务**：如使用 [whisper-asr-webservice](https://github.com/ahmetoner/openai-whisper-asr-webservice) 部署到本地/服务器
- **任何兼容 `/v1/audio/transcriptions` 的 API**：只要请求格式与 OpenAI 一致即可
- **注意**：如果一个服务只提供聊天模型，不提供音频转写接口，则不能直接用于 ASR

### Q8：软件会偷偷上传我的数据吗？

- **不会**。所有 API Key 仅保存在本地 `config/user.json` 中。
- 语音数据只发送给你配置的 ASR 服务提供商（讯飞或你自建的 HTTP 服务）。
- 软件本身没有自己的服务器，不会收集任何数据。

### Q9：必须使用虚拟环境吗？

- **强烈建议使用**。虚拟环境能避免依赖包与系统 Python 冲突，也方便以后卸载/迁移。
- 如果你确定系统 Python 很干净，也可以直接 `pip install -r requirements.txt`，但出了问题较难排查。

### Q10：venv 和 uv 有什么区别？选哪个？

| 对比项 | venv | uv |
|--------|------|-----|
| 安装速度 | 正常 | **极快**（10~100倍） |
| 学习成本 | 低（Python 内置） | 低（一条命令即可） |
| 是否需要额外安装 | 否 | 需安装 uv 工具 |
| 兼容性 | 标准 Python | 与 venv 完全兼容 |
| 推荐人群 | 大多数用户 | 追求速度、经常装包的用户 |

**结论**：不想折腾选 **venv**，想体验飞快速度选 **uv**。

## 8. 故障排查

### 日志查看

运行程序时，终端会输出日志。若遇到问题，请复制终端中的错误信息。

常见日志级别：
- `[INFO]` — 正常状态变更（开始、停止、连接成功）
- `[WARNING]` — 可恢复异常（短暂断网、重连中）
- `[ERROR]` — 需要处理的错误（鉴权失败、设备不可用）

### 虚拟环境排查

若运行时报 `ModuleNotFoundError` 或找不到包，大概率是**虚拟环境未激活**或激活错了环境：

```bash
# venv 用户：每次运行前必须激活
source venv/bin/activate        # Linux/macOS
venv\Scripts\activate.bat      # Windows CMD

# 验证是否在虚拟环境中
which python   # 应显示 venv/bin/python（Linux/macOS）
where python   # 应显示 venv\Scripts\python.exe（Windows）
```

**uv 用户**：
```bash
# 确保使用 uv run，它会自动找到 .venv
uv run python main.py
# 不要直接 python main.py，否则可能用到系统 Python
```

### 麦克风检测

若怀疑麦克风有问题，可在终端运行（确保虚拟环境已激活）：

```bash
cd AIAnswer
python -c "
import sys; sys.path.insert(0, 'src')
from audio.capture import AudioCapture
for d in AudioCapture.list_devices():
    print(d)
"
```

查看输出中是否有你的麦克风设备。若没有，请检查系统麦克风权限设置。

### 重置配置

若配置混乱导致无法启动，可删除用户配置文件，下次启动会自动重建：

```bash
rm config/user.json
```

## 9. 隐私与安全

- **语音数据**：
  - 使用讯飞时，实时发送至讯飞开放平台进行识别。
  - 使用 http_asr 时，发送至你指定的 API Base 地址。
  - 本地不会长期保存录音文件。
- **API Key**：
  - 仅保存在本地 `config/user.json` 中。
  - 不会上传至任何第三方服务器。
- **AI 上下文**：
  - 每次提问的上下文仅在内存中暂存，回答完成后即释放。
  - 不会长期存储对话历史。
