# AI课堂助手 — 模块详细设计

## 1. 全局数据流与队列设计

程序采用 **生产者-消费者** 模式，通过 `asyncio.Queue` 解耦各模块：

```python
# 全局队列（在 main.py 中创建并注入各模块）
audio_queue: asyncio.Queue[bytes]      # AudioCapture → StreamingASR
asr_text_queue: asyncio.Queue[str]     # StreamingASR → TextBuffer → UI
trigger_queue: asyncio.Queue[dict]     # TriggerDetector → LLMClient
llm_text_queue: asyncio.Queue[str]     # LLMClient → UI
```

所有模块共享同一个 `asyncio` 事件循环，避免线程安全问题。

## 2. 模块接口定义

### 2.1 AudioCapture (音频采集)

**职责**：从麦克风持续读取音频chunk，推入队列。

```python
class AudioCapture:
    def __init__(self, queue: asyncio.Queue[bytes], 
                 sample_rate: int = 16000, 
                 chunk_duration_ms: int = 200,
                 device_index: Optional[int] = None):
        """
        queue: 音频bytes队列
        sample_rate: 采样率，讯飞ASR要求16kHz
        chunk_duration_ms: 每个chunk的时长，默认200ms
        device_index: 麦克风设备索引，None表示默认设备
        """
    
    async def start(self) -> None:
        """启动采集循环，持续向queue放入音频chunk"""
    
    async def stop(self) -> None:
        """停止采集，释放pyaudio资源"""
    
    @staticmethod
    def list_devices() -> list[dict]:
        """列出所有可用音频输入设备，返回[{index, name}]"""
```

**关键参数**：
- 采样率固定 16000 Hz（单声道，16bit），这是讯飞ASR的标准要求
- chunk大小 = `sample_rate * 2 bytes * chunk_duration_ms / 1000` ≈ 6400 bytes (200ms)
- pyaudio的read操作是阻塞的，需在单独的线程中执行，通过 `asyncio.to_thread` 或 `loop.run_in_executor` 桥接到异步

**状态**：`IDLE` → `RUNNING` → `STOPPED`

### 2.2 StreamingASR (讯飞流式语音识别)

**职责**：通过WebSocket连接讯飞ASR服务，实时发送音频并接收识别文本。

```python
class StreamingASR:
    def __init__(self, 
                 audio_queue: asyncio.Queue[bytes],
                 text_queue: asyncio.Queue[str],
                 app_id: str,
                 api_key: str,
                 api_secret: str):
        """
        audio_queue: 从AudioCapture接收音频
        text_queue: 输出识别文本片段
        app_id/api_key/api_secret: 讯飞开放平台鉴权信息
        """
    
    async def start(self) -> None:
        """
        建立WebSocket连接，启动两个协程：
        1. sender: 从audio_queue取chunk发送给讯飞
        2. receiver: 接收讯飞返回的JSON，解析文本推入text_queue
        """
    
    async def stop(self) -> None:
        """发送结束标记，关闭WebSocket，清理资源"""
```

**讯飞WebSocket协议要点**：
- 鉴权：通过 `api_key` + `api_secret` 生成 HMAC-SHA256 签名，拼接在URL参数中
- 音频格式：raw PCM, 16000Hz, 16bit, 单声道，小端序
- 数据帧：每次发送一个音频chunk，最后一帧需标记 `status = 2`（结束）
- 返回字段：
  - `data.result.ws`：词级结果列表
  - `data.result.pgs`：`apd`（追加）或 `rpl`（替换），流式结果会修正前面的文本
  - 需处理文本替换/追加逻辑，拼接成完整句子

**输出到 text_queue 的文本粒度**：
- 每次返回完整的当前句（不是增量字），这样下游检测更简单
- 例如：收到 "下面我来问个问题"

**错误处理**：
- WebSocket断连：自动重连3次，超过则报错并停止程序
- 鉴权失败：立即抛出异常，提示用户检查配置

### 2.3 TextBuffer (文本滑动窗口)

**职责**：维护最近的ASR文本历史，供提问检测使用。

```python
class TextBuffer:
    def __init__(self, max_age_seconds: float = 60.0):
        """
        max_age_seconds: 保留最近N秒的文本
        """
    
    def append(self, text: str, timestamp: Optional[float] = None) -> None:
        """追加新文本片段，自动清理过期内容"""
    
    def get_context(self, tail_sentences: int = 2) -> str:
        """
        获取缓冲区末尾的N个句子作为上下文。
        返回：str，用于LLM输入
        """
    
    def get_full_text(self) -> str:
        """获取当前缓冲区的完整文本（用于保存记录）"""
```

**实现细节**：
- 内部用列表存储 `(timestamp, text)` 元组
- 按句号 `。！？.!?` 分句
- `get_context` 提取最后2个完整句子（当前句 + 前一句），避免上下文过长

### 2.4 TriggerDetector (提问检测器)

**职责**：扫描文本，检测疑似提问的语句。

```python
class TriggerDetector:
    def __init__(self, 
                 text_buffer: TextBuffer,
                 trigger_queue: asyncio.Queue[dict],
                 triggers: Optional[list[str]] = None,
                 cooldown_seconds: float = 5.0):
        """
        text_buffer: 文本缓冲区
        trigger_queue: 命中后输出上下文到此队列
        triggers: 触发词列表，默认使用内置词库
        cooldown_seconds: 冷却时间，避免同一提问重复触发
        """
    
    async def start(self, text_queue: asyncio.Queue[str]) -> None:
        """
        持续从text_queue读取新文本，更新buffer并检测。
        命中后，将上下文dict推入trigger_queue。
        """
    
    def stop(self) -> None:
        """停止检测"""
```

**输出格式（推入trigger_queue）**：
```python
{
    "type": "question_triggered",
    "context": "老师说的最后两句话文本",
    "timestamp": 1715000000.0,
    "matched_trigger": "问个问题"
}
```

**检测逻辑**：
1. 接收到新文本 → 追加到 TextBuffer
2. 用当前文本末尾匹配触发词（`in` 或正则）
3. 若命中且距离上次触发超过 `cooldown_seconds`：
   - 调用 `text_buffer.get_context()` 提取上下文
   - 推入 `trigger_queue`

**触发词匹配策略**：
- 精确包含匹配：`if trigger in text`
- 支持正则扩展（如 `r"问.{0,2}问题"`）
- 未来可扩展为轻量分类模型，当前用规则足够

### 2.5 LLMClient (LLM流式客户端)

**职责**：接收提问上下文，调用LLM API，流式返回结果。

```python
class LLMClient:
    def __init__(self,
                 trigger_queue: asyncio.Queue[dict],
                 output_queue: asyncio.Queue[str],
                 api_base: str = "https://api.openai.com/v1",
                 api_key: str = "",
                 model: str = "gpt-4o-mini",
                 system_prompt: str = "回答该提问，口语化并简短精炼"):
        """
        trigger_queue: 从TriggerDetector接收提问上下文
        output_queue: 输出AI回答文本片段（SSE增量）
        """
    
    async def start(self) -> None:
        """
        持续从trigger_queue取任务，发送HTTP SSE请求，
        将流式返回的文本推入output_queue。
        """
    
    def stop(self) -> None:
        """停止处理新任务"""
```

**请求格式（OpenAI兼容）**：
```json
{
  "model": "gpt-4o-mini",
  "messages": [
    {"role": "system", "content": "回答该提问，口语化并简短精炼"},
    {"role": "user", "content": "老师的提问上下文文本"}
  ],
  "stream": true,
  "max_tokens": 150,
  "temperature": 0.7
}
```

**SSE解析**：
- 逐行读取 `data: {...}`
- 提取 `choices[0].delta.content`
- 遇到 `[DONE]` 标记结束
- 每个字/词立即推入 `output_queue`

**错误处理**：
- API 401/403：提示Key错误
- 超时：5秒无首字响应则提示网络或模型繁忙
- 其他异常：捕获并记录，不影响主程序运行

### 2.6 UI (tkinter界面)

**职责**：展示实时字幕、AI回答、设置面板。

```python
class MainWindow:
    def __init__(self, 
                 on_start: Callable,
                 on_stop: Callable,
                 on_config_save: Callable):
        """
        on_start: 点击"开始听课"回调
        on_stop: 点击"停止"回调
        on_config_save: 保存配置回调
        """
    
    def append_subtitle(self, text: str) -> None:
        """追加实时字幕文本"""
    
    def append_ai_answer(self, text: str, is_new: bool = False) -> None:
        """
        追加或更新AI回答文本。
        is_new=True: 这是一个新回答，另起一块区域
        is_new=False: 追加到当前回答末尾（流式更新）
        """
    
    def set_status(self, text: str) -> None:
        """更新底部状态栏"""
    
    def show_settings(self) -> None:
        """弹出设置对话框"""
    
    def run(self) -> None:
        """启动tkinter主循环（需在主线程运行）"""
```

**界面布局**：
```
+--------------------------------------------------+
|  AI课堂助手                    [设置] [开始/停止]  |
+--------------------------------------------------+
|  实时字幕                                        |
|  +--------------------------------------------+  |
|  | 老师：下面我来问个问题...                     |  |
|  | 老师：请一位同学回答一下这个概念的用法...       |  |
|  +--------------------------------------------+  |
|                                                  |
|  AI 回答                                         |
|  +--------------------------------------------+  |
|  | [12:34] 这个问题可以从实际应用场景来理解...    |  |
|  |         简单来说就是...                        |  |
|  +--------------------------------------------+  |
|                                                  |
|  状态：运行中 | ASR: 已连接 | LLM: 就绪           |
+--------------------------------------------------+
```

**线程安全**：
- tkinter 必须在主线程运行
- 其他模块在 asyncio 事件循环中运行
- 通过 `root.after()` 或 `asyncio.run_coroutine_threadsafe()` 进行跨线程通信

## 3. 状态机设计

### 3.1 应用全局状态

```
IDLE ──[点击开始]──→ CONNECTING ──[ASR连接成功]──→ LISTENING
  ↑                                                      │
  └────────────────[点击停止]──────────────────────────────┘
```

| 状态 | 含义 |
|------|------|
| IDLE | 初始状态，未开始采集 |
| CONNECTING | 正在连接讯飞WebSocket |
| LISTENING | 正常听课中，全链路运行 |
| ERROR | 发生错误，显示错误信息 |

### 3.2 ASR连接状态（StreamingASR内部）

```
DISCONNECTED ──[start()]──→ CONNECTING ──[on_open]──→ STREAMING
     ↑                                                    │
     └──────────[异常/ stop()]─────────────────────────────┘
```

## 4. 错误与降级策略

| 场景 | 处理方式 |
|------|----------|
| 麦克风无法打开 | 弹窗提示用户检查麦克风权限和设备 |
| ASR WebSocket断开 | 自动重连3次，失败后回IDLE状态并提示 |
| ASR鉴权失败 | 立即停止，提示检查讯飞Key |
| LLM API超时(5s) | 提示"AI回答超时，请检查网络或模型" |
| LLM API 401 | 提示"API Key无效" |
| LLM返回空内容 | 忽略，不展示 |
| 未知异常 | 捕获并记录日志，不崩溃主程序 |

---

## 5. 日志规范

使用 Python 标准库 `logging`，级别规范：

- **INFO**：状态变更（开始听课、停止、ASR连接成功、检测到提问、AI回答完成）
- **DEBUG**：详细数据流（收到音频chunk、ASR返回文本、SSE收到token）
- **WARNING**：可恢复异常（ASR短暂断开正在重连）
- **ERROR**：需用户介入的异常（鉴权失败、设备不可用）

日志格式：`[%(asctime)s] [%(levelname)s] [%(name)s] %(message)s`

## 6. 关键时序图

### 6.1 正常提问-回答流程

```
AudioCapture    StreamingASR    TextBuffer+Detector    LLMClient       UI
     │                │                │                │             │
     │── audio ─────→│                │                │             │
     │                │── text ──────→│                │             │
     │                │                │ append text    │             │
     │                │                │ no trigger     │             │
     │                │                │─── update subtitle ────────→│
     │── audio ─────→│                │                │             │
     │                │── text ──────→│                │             │
     │                │                │ trigger hit!   │             │
     │                │                │── context ────→│             │
     │                │                │                │ call API    │
     │                │                │                │── SSE ─────→│
     │                │                │                │  token      │
     │                │                │                │ stream ────→│
     │                │                │                │ [DONE]      │
```

### 6.2 程序启动时序

```
main.py
  │
  ├── 加载配置 (ConfigManager)
  ├── 初始化UI (MainWindow)
  ├── 注册回调：on_start() / on_stop()
  ├── 启动 asyncio 事件循环（在后台线程）
  └── 进入 tkinter mainloop()

on_start() 被点击时：
  ├── 创建 asyncio.Queue (audio, text, trigger, llm)
  ├── 初始化各模块并注入queue
  ├── 创建并启动协程：AudioCapture, StreamingASR, TriggerDetector, LLMClient
  └── UI状态更新为 LISTENING
```
