# AI课堂助手 — 开发计划与里程碑

## 1. 版本规划

### V1.0 MVP（最小可用版本）
目标：实现核心闭环——录音 → ASR → 检测 → LLM → 展示

### V1.1 体验优化
目标：降低延迟、提升稳定性、增加配置灵活性

### V1.2 功能增强
目标：记录保存、多ASR提供商、更智能的检测

## 2. 任务拆解与优先级

### Phase 2：音频与ASR模块（预计 2-3 天）

| 任务 | 文件 | 优先级 | 说明 |

## 3. 风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| 讯飞WebSocket协议复杂 | 延迟 | 预留充足时间，参考官方Demo |
| pyaudio在不同系统兼容性 | 中 | 优先保证Linux/Windows，Mac次之 |
| tkinter跨线程更新易出错 | 中 | 严格使用 `root.after()` 做UI更新 |
| 上课环境网络不稳定 | 中 | 增加ASR重连机制和LLM超时处理 |
| 课堂噪音导致ASR准确率下降 | 低 | 这是ASR服务商的问题，用户可选择近场麦克风 |

## 4. 开发顺序建议

推荐按以下顺序开发，每步可独立测试：

```
Week 1
├── Day 1-2: Phase 1 (框架 + 配置) + Phase 2 (音频采集)
├── Day 3-4: Phase 2 (讯飞ASR接入)
├── Day 5:   Phase 3 (文本缓冲 + 提问检测)

Week 2
├── Day 1-2: Phase 3 (LLM客户端)
├── Day 3-4: Phase 4 (UI界面)
├── Day 5:   Phase 5 (联调 + 修bug)

Week 3
├── Day 1-2: Phase 6 (文档 + 打包 + 优化)
```

---

## 5. 立即开始的下一步

若确认开发，建议立即执行：

1. **申请讯飞开放平台账号**（https://www.xfyun.cn/）
   - 创建"语音听写"应用，获取 `APPID`、`APIKey`、`APISecret`
   - 确认有免费额度（新用户通常足够）

2. **准备LLM API Key**
   - 建议先用 DeepSeek / 通义千问 / OpenAI 等任一支持OpenAI格式的API

3. **编码启动**
   - 按 Phase 1 → Phase 2 顺序实现
   - 每完成一个模块即进行单元测试

## 6. 未来扩展（V1.1+）

- **多ASR提供商**：支持阿里云、腾讯云ASR作为备选
- **智能检测升级**：用轻量分类模型替代关键词（如text2text的zero-shot）
- **课堂笔记导出**：一键导出完整字幕为Markdown
- **快捷键支持**：全局热键开始/停止，无需切换窗口
- **回答语音播报**：AI回答生成后可选TTS朗读
